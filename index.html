03API 文档API Docs

主要面向开发者和 Agent 直接调用，支持匿名上载、并行上载、状态查询、静态站点目录发布，或携带用户 API Key 发布文件。Primarily for developers and Agents, supporting anonymous uploads, parallel uploads, status queries, static-site folder publishing, or publishing with user API Keys.

最简接入Quick Start 
# 文件最小链路：prepare -> PUT -> complete
# 1) 申请上载地址。匿名可不传 apiKey；如需更小分片，可传 preferredPartSize。
curl -X POST "https://www.okfile.com/api/upload/prepare" \
  -H "Content-Type: application/json" \
  --data '{"filename":"photo.jpg","size":12345,"contentType":"image/jpeg","preferredPartSize":5242880}'

# 2) 对 prepare 返回的 uploadUrl 或 parts[].uploadUrl 执行 PUT
# 3) 完成后仅用 id 调 complete。若返回 missingParts，只重传缺失分片即可。
curl -X POST "https://www.okfile.com/api/upload/complete" \
  -H "Content-Type: application/json" \
  --data '{"id":"a3k7m92x"}'

# 可选：用 status 查询分片进度
curl "https://www.okfile.com/api/upload/status/a3k7m92x"

# 站点最小链路：site/prepare -> 文件上载 -> site/complete
# 如果没有根目录 index.html，不要额外生成它；OkFile 会自动渲染目录列表
curl -X POST "https://www.okfile.com/api/site/prepare" \
  -H "Content-Type: application/json" \
  --data '{"siteName":"docs-site","files":[{"path":"docs/getting-started.md","size":1200,"contentType":"text/markdown; charset=utf-8"},{"path":"assets/app.css","size":3200,"contentType":"text/css; charset=utf-8"},{"path":"images/logo.png","size":4200,"contentType":"image/png"}]}'

# 然后按普通文件上载链路上传每个文件，收集 fileId，再调用：
curl -X POST "https://www.okfile.com/api/site/complete" \
  -H "Content-Type: application/json" \
  --data '{"siteId":"st_xxxx","siteToken":"token_xxxx","files":[{"relativePath":"docs/getting-started.md","fileId":"a3k7m92x"},{"relativePath":"assets/app.css","fileId":"b8f2k19a"},{"relativePath":"images/logo.png","fileId":"c9d0e1f2"}]}'
# File minimal flow: prepare -> PUT -> complete
# 1) Request upload URLs. Omit apiKey for anonymous mode; pass preferredPartSize if you want smaller parts.
curl -X POST "https://www.okfile.com/api/upload/prepare" \
  -H "Content-Type: application/json" \
  --data '{"filename":"photo.jpg","size":12345,"contentType":"image/jpeg","preferredPartSize":5242880}'

# 2) PUT to uploadUrl or each parts[].uploadUrl returned by prepare
# 3) Call complete with only id. If missingParts is returned, re-upload only those parts.
curl -X POST "https://www.okfile.com/api/upload/complete" \
  -H "Content-Type: application/json" \
  --data '{"id":"a3k7m92x"}'

# Optional: query multipart progress with status
curl "https://www.okfile.com/api/upload/status/a3k7m92x"

# Site minimal flow: site/prepare -> upload files -> site/complete
# If there is no root index.html, do not generate one; OkFile renders the directory listing automatically
curl -X POST "https://www.okfile.com/api/site/prepare" \
  -H "Content-Type: application/json" \
  --data '{"siteName":"docs-site","files":[{"path":"docs/getting-started.md","size":1200,"contentType":"text/markdown; charset=utf-8"},{"path":"assets/app.css","size":3200,"contentType":"text/css; charset=utf-8"},{"path":"images/logo.png","size":4200,"contentType":"image/png"}]}'

# Then upload each file through the normal file flow, collect fileId values, and call:
curl -X POST "https://www.okfile.com/api/site/complete" \
  -H "Content-Type: application/json" \
  --data '{"siteId":"st_xxxx","siteToken":"token_xxxx","files":[{"relativePath":"docs/getting-started.md","fileId":"a3k7m92x"},{"relativePath":"assets/app.css","fileId":"b8f2k19a"},{"relativePath":"images/logo.png","fileId":"c9d0e1f2"}]}'

PUT uploadUrl / parts[].uploadUrl 
# 2) 对 prepare 返回的 uploadUrl 执行 PUT
# 单文件模式：PUT 一次即可
curl -X PUT "$uploadUrl" \
  -H "Content-Type: image/jpeg" \
  -H "Content-Length: 12345" \
  --data-binary "@photo.jpg"

# 分片模式：对每个 parts[i].uploadUrl 分别 PUT
# 多文件任务可并行执行多个 PUT
# 2) PUT the file body to the uploadUrl returned by prepare
# Single-upload mode: one PUT is enough
curl -X PUT "$uploadUrl" \
  -H "Content-Type: image/jpeg" \
  -H "Content-Length: 12345" \
  --data-binary "@photo.jpg"

# Multipart mode: PUT to each parts[i].uploadUrl separately
# Multiple file tasks can run multiple PUTs in parallel
POST /api/upload/complete 
# 3) 上载完成后确认

方法Method端点Endpoint说明Description
POST/api/upload/prepare申请单传或分片上载地址；可选携带 `apiKey` 和 `preferredPartSize`，返回 {mode, uploadUrl/parts, id, url, playUrl, type}Request single/multipart upload URL; optionally carry `apiKey` and `preferredPartSize`, returns {mode, uploadUrl/parts, id, url, playUrl, type}
PUTuploadUrl / parts[].uploadUrl把文件内容或分片直接写入 prepare 返回的动态上载地址Write file content or part directly to dynamic upload URL returned by prepare
POST/api/upload/complete确认文件已上载；单传和分片都只需 id，分片由后台自动合并Confirm file uploaded; both modes only need id, multipart merged automatically by backend
GET/api/upload/status/{id}查询分片上载任务的进度与状态Query progress and status of a multipart upload task
POST/api/site/prepare为静态站点目录创建发布会话，提交站点名、可选入口文件和完整文件清单；没有根 `index.html` 时不要额外生成它，直接省略 `entryPath` 即可Create a publish session for a static site folder with the site name, optional entry file, and full file manifest; if there is no root `index.html`, do not generate one and simply omit `entryPath`
POST/api/site/complete提交 {relativePath, fileId} 映射完成站点发布，子目录路径会保留，返回 {siteUrl, entryUrl}Submit the {relativePath, fileId} mapping to finalize site publishing, preserving nested subdirectory paths, and return {siteUrl, entryUrl}
GET/i/{id}默认文件直链，直接返回原始文件Default file direct link, returns original file directly
GET/i/{id}?play=1预览/播放页面（图片、视频、PDF）Preview/Playback page (Image, Video, PDF)
