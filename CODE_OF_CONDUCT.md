# Code of Conduct

## Our Standard

We want OkFile to be a respectful, constructive, and practical project space.

Examples of positive behavior:

- being respectful in issues and pull requests
- giving actionable technical feedback
- assuming good intent
- keeping discussions focused on the code, docs, and user impact

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- sharing secrets, private data, or exploit details irresponsibly
- deliberately disruptive or bad-faith behavior

## Enforcement

Project maintainers may remove comments, close discussions, or limit participation when behavior harms the project or its contributors.

## Scope

This applies to:

- GitHub issues
- pull requests
- code review discussions
- repository-linked project spaces

## Reporting

If you encounter abusive behavior or unsafe disclosure, contact the project maintainer through a private channel.
